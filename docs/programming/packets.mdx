# Telemetry Packet Format

Packets are used to communicate between the rocket and the ground station. They convey information about the state of the rocket, alert the ground station of any updates, and enable the ground station to make adjustments to the rocket before launch.

*Important Note*: Packets can only be sent to the rocket BEFORE launch, not AFTER. This is to comply with regulations.

## Packets FROM ground software TO flight software

|Header                              |Message Format     |
|------------------------------------|-------------------|
|set_abort_level (To be implemented) | <code>Level: AbortLevel</code>|
|soft_abort (deprecated)             |                   |
|undo_soft_abort (deprecated)        |                   |
|progress                            |                   |
|solenoid_actuate                    | <code>valve_type: X,<br/>valve_location: X,<br/>actuation_type: X,<br/>actuation_priority: X</code>|
|valve_request                       | <code>valve_type: X,<br/>valve_location: X</code>    |
|sensor_request                      | <code>sensor_type: X,<br/>sensor_location: X</code>  |

## Packets FROM flight software TO ground software

| Header                   | Message Format                                                                                                                  |
|--------------------------|---------------------------------------------------------------------------------------------------------------------------------|
| control_task_start       | <code>task: stage \| pressure \| telemetry \| valve \| sensor</code>                                                                       |
| stage_status             | <code>stage: X<br>status: X</code>                                                                                                         |
| abort_level              | <code>level: AbortLevel</code>                                                                                                             |
| abort_request_response   | <code>successful: boolean<br>level: AbortLevel</code>                                                                                      |
| heartbeat                | <code>timestamp: X</code>                                                                                                                  |
| sensor_data              | <code>[type]: {<br>    [location]: {<br>        measured_value: X<br>        normalized_value: X<br>        status: X<br>    }<br>}</code> |
| abort_alert              | <code>level: none \| soft \| hard<br>dangerous_sensors?: [<br>    type: X<br>    location: X<br>    normalized_value: X<br>]</code>        |
| valve_actuation_response | <code>type: X<br>location: X<br>found: boolean<br>successful: boolean</code>                                                               |
| valve_request_response   | <code>type: X<br>location: X<br>found: boolean<br>actuation_type?: X<br>actuation_priority?: X</code>                                      |
| valve_data               | <code>[type]: {<br>    [location]: {<br>        actuation_type: X<br>        actuation_priority: X<br>    }<br>}</code>                    |
| valve_actuated           | <code>type: X<br>location: X</code>                                                                                                        |
